# Documentation System

Este documento define una estructura general para no mezclar tipos de documentacion.

## Cuatro Capas

### 1. Metodologia

Responde a:

- como conviene trabajar
- como colaborar
- como verificar

### 2. Estado

Responde a:

- donde esta el proyecto hoy
- que esta hecho
- que esta pendiente

### 3. Direccion

Responde a:

- hacia donde debe ir el proyecto
- que fases o bloques existen
- que prioridades mandan

### 4. Bitacora

Responde a:

- que ha pasado
- que problemas surgieron
- que decisiones se tomaron
- que verificacion se hizo

## Reglas Generales

- el roadmap no sustituye al progreso
- el progreso no sustituye a la bitacora
- la metodologia no sustituye a las reglas operativas del repo
- las notas externas no sustituyen a ninguna de las capas anteriores

## Minimos Recomendados

En casi cualquier proyecto conviene tener:

- `README.md`
- `AGENTS.md` o equivalente
- plan o roadmap
- documento de progreso actual
- bitacora de iteraciones
- carpeta de metodologia reusable

## Regla De Actualizacion

Si cambia:

- el proceso general reusable -> actualizar metodologia
- el estado -> actualizar progreso
- la historia del trabajo -> actualizar bitacora
- la direccion -> actualizar plan

## Frecuencia De Actualizacion

La metodologia general debe ser estable.

- no se toca por cada cambio
- no se usa como bitacora
- no guarda detalles de stack
- solo recibe reglas que sirvan fuera del proyecto actual
